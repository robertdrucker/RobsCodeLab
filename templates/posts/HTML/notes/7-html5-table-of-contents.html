Summary

An often overlooked aspect of good web design is ensuring that each page has a proper HTML structure. In this post, we take a look at a simple Table of Contents tool that can be used to check your page hierarchy from the point of view of a screen reader. In addition, a few “best-practice” rules are reviewed for ensuring a good page layout, in terms of both structure and semantics.

Key Words

  • structural element, sectioning element, table of contents, HTML5 Outliner

Notes

  • Web crawlers and assistive technologies rely on the structure of your web page to analyze and characterize it.
  • Unorganized markup code will confuse screen readers and other assistive technologies. A poor structure can also make it diffcult on web crawlers and robots and hurt your SEO.
  • There are tools available that can used to generate a Table of Contents for any website page. Checking the Table of Contents is a must for verifying that the structure of your code is as intended.
  • A tool I generally use is the HTML5 Outliner (h5o) Google Chrome extension. In addition to generating a Table of Contents for your page, this tool checks for the presence of headings in each sectioning element and will warn you if any are missing.
  • There are five sectional elements in HTML5: section, nav, article, and aside. Recall that it is considered a best practice to always include a heading in each sectioning element. The body, which is called a “sectioning root”, should also contain a heading.
  • Another good rule to follow is to consider using structural element instead of a div element. This should be done when the enclosed content can better be classified by a structural element. For example, an article should be used to embed a blog post.
  • HTML5 structural elements include: header, nav, main, section, article, figure, and footer.
  • Note that each of five sectional elements are also structural elements, but not all structural elemments are sectional elements.
