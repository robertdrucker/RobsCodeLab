Summary

This post is the first of a two-part series about the Python generator. Here, the characteristics of a generator are reviewed, and we see how to create a generator function using a yield statement. Additionally, we take a look at how a for loop, under the hood, iterates over a generator.

Key Words

  • generator, yield, iterable, iterator protocol, lazy iterator

Notes

  • You have seen what a generator does. It is invoked; it runs until a yield statement is executed; it returns the value of an expression in the yield statement; and it moves on to the next iteration when a next statement is executed.
  • But, how is a custom generator constructed? The simple way is to just incorporate a yield statement into a function. This is what is done in the accompanying code.
  • Or, you can get more elaborate and build a generator from scratch using a class (we'll do this in Part 2).
  • Any function definition with a yield statement is a generator.
  • A generator returns an iterator object with a sequence of values.
  • The yield statement halts the generator and returns a value through an expression.
  • Generators are iterable, but not all iterables are generators. For example, the range function is a generator. The list object is iterable, but it is not a generator.
  • Since a generator is an iterable, it can be looped over. However, where as a list stores all of its values in memory, a generator only computes and stores a single value at a time. For this reason, we say that a generator returns a lazy iterator.
  • Because of their unique properties, a generator generally runs fast and is memory efficient.
