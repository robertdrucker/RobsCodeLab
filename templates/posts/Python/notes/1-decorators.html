Summary

The code in this post utilizes a decorator to “prettify” a function that compares the output of three different trigonometric functions across a given set of values. The example given here also provides a good illustration of how the unpacking operator * is used.

Key Words

  • Decorator
  • Unpacking operator

Notes

  • You can think of the decorator as a design pattern in Python that allows us to wrap a function and enhance its behavior without changing it. You can see this in the included programming code.
  • Pay attention to how the decorated function, trig_func_comparison, gets sandwiched in a wrapper that has supplementary print statements.
  • Also, note how variable passing is accomplished with the unpacking operator. Concentrate on *args, here. This is for positional arguments. The other one, *kwargs, is for keyword arguments, and there are none in the example code.
  • I included *kwargs as an argument to the wrapper function only to be consistent with the decorator design pattern commonly used.
  • Their are many uses of decorators. For example, a decorator can be used to compare the relative processing time of two different functions. Frameworks, such as Flask and Django, also use decorators for such tasks as user authentication and routing.

Definitions

  • A decorator is a function that accepts another function as an argument, wraps the input function, and essentially injects additional functionality into it.
