Summary

There is quite a bit to know about the rest parameter and the spread syntax. Rather than dive into all use cases, here the basic functionality behind each operator is reviewed. Thus, the content below covers sufficient ground to understand typical usage of the spread and rest operators in JavaScript applications.

Key Words

  • spread operator
  • rest operator
  • formal parameter
  • actual parameter

Notes

  • Both the spread and the rest operators were introduced in ES6. Each is denoted by three consecutive dots(...), and the meaning of these three dots depends upon the context.
  • As shown in the accompanying code, when used inside of an array, the spread operator is used to extract all elements from another array in a clean fashion. Likewise, when used inside of an object, the spread operator is used to extract all properties from another object.
  • In contrast, the rest operator, or rest parameter as it is sometimes called, is used as a formal parameter to a function. In this context, it merges a list of actual parameters inside of function call
  • All of this is best understood by looking at a few examples. So, let's dive in.
